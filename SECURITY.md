# Security policy

## Supported versions

We support the latest minor release on the `0.x` line. Once `1.0.0` ships, the latest two minor releases are supported.

## Reporting a vulnerability

Please report security issues privately, not as a public GitHub issue.

- Preferred: open a [GitHub private security advisory](https://github.com/spec-0/CLI/security/advisories/new).
- Or email **security@spec0.io**.

We aim to acknowledge reports within **48 hours** and provide a remediation timeline within **5 business days**.

When reporting, please include:

- Affected CLI version (`spec0 --version`).
- Reproduction steps or proof-of-concept.
- Impact assessment (what an attacker could achieve).

## Disclosure

We follow coordinated disclosure: we ask reporters to delay public disclosure until a fix is released, typically within 90 days. Credit is given in the changelog unless you request anonymity.
